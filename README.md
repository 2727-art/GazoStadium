# 貼り合いスタジアム

好きな画像を5枚用意し、知らない相手と魅力を競うオンライン画像対戦ゲームです。1～10点で戦う1on1・2on2と、順位投票で生き残る4人バトルロワイヤルを遊べます。

- Firebaseで匿名ログイン・ランダムマッチング・対戦同期・チャットを行い、画像本体はWebRTC DataChannelで対戦相手へ直接送信
- トップ画面の「戦略型1on1対戦」から、匿名自己紹介を読んで相手対策デッキを組むオンライン対戦を遊べます
- トップ画面の参加状況は、通常型1on1・戦略型1on1・2on2・バトルロワイヤルを別々に表示します
- トップ画面の参加状況はRealtime Databaseを購読し、待機・対戦状態の変化をリアルタイムで更新します
- オンライン総合ランキングは4モードの結果を合算し、デイリー・ウィークリー・マンスリーの選択中期間だけREST APIから取得します
- ランキング参加設定は4モード共通で、各モードの準備画面とランキング画面から切り替えられます
- 任意のプロフィール画像を同一ブラウザのIndexedDBへ保存し、対戦中だけWebRTCで参加者へ直接転送します
- ランキングコメントも欄を開いた時だけ取得し、投稿・削除時だけ匿名認証付きREST APIへ接続します

## 起動方法

ES Modulesを使うため、ローカルサーバーから開きます。ビルドやパッケージのインストールは不要です。

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

その後、`http://127.0.0.1:4173/`を開きます。オンライン対戦の動作確認は、1on1なら2環境、2on2・バトルロワイヤルなら4環境のように、保存領域の異なるブラウザまたはブラウザプロファイルで行ってください。

## 対戦ルール

### 戦略型1on1対戦（オンライン）

Firebaseの専用待機キューで知らない相手とマッチングし、匿名自己紹介から相手の好みを推測してデッキを構築する2人用オンラインモードです。

- 各プレイヤーは名前、弱点候補3つ、追撃セリフを先に登録し、本当の弱点1つとブラフ2つを決める
- 追撃セリフは定型文または自由記述を選択。自由記述は1行・最大40文字で、空白時は定型文へ戻る
- 弱点は乱数付きSHA-256コミットとして先に共有し、回答確定後の公開時に改変されていないことを照合
- マッチ成立後は相手の名前を伏せたまま3つの弱点候補を表示し、対戦を受けるか撤退するかを選択
- 匿名紹介からデッキ封印までは名前・プロフィール画像を伏せた専用チャットで質問やブラフを交わし、弱点推理の材料にできる
- 対戦を受けたプレイヤーは、メイン5枚と任意のリザーブ最大5枚を相手の好みに合わせて構築
- 各画像には任意で音声を1つ添付可能。ブラウザ内で先頭10秒までを22.05kHz・モノラル・16bit WAVへ変換し、音量を正規化する
- 追撃用に音声内の3秒区間を指定でき、弱点看破後の連続追撃では最大3枚・合計最大9秒を順番に再生
- 両者のデッキ封印後にプレイヤーネームを公開
- 名前公開後も全対戦フェイズでチャットを継続し、参加者欄と発言横にP2P受信済みのプロフィール画像を表示
- 最大5ラウンド。両者がメイン画像を秘密選択し、同時公開後に相手の画像を1～10点で秘密採点
- 採点とは別に、各プレイヤーは1～3点時の「もう1枚見せろ」を2回、9～10点時の「追撃を許可」を1回まで使用可能
- 追加要求を受けた側はリザーブを1枚提示し、メインと追加画像の高い方をラウンドの画像パワーに採用
- 追撃側は事前登録セリフとリザーブを公開し、追撃画像の採点の半分をメインの画像パワーへ加算
- 追加画像から追加要求・追撃は連鎖せず、両者の画像パワー確定後に高い側が同値のダメージを与える
- 各ラウンドの結果後、どちらかが一度だけ早期看破を宣言可能。宣言すると両者の秘密回答フェイズが始まり、未宣言ならROUND 3後に強制開始
- 両者の回答確定後に弱点を公開してコミットを検証。看破失敗者は自分に5ダメージを受け、再回答できない
- 看破成功者は残っているリザーブから最大3枚を選び、`4 → 3 → 2`の固定ダメージで連続追撃。両者成功時は同時解決
- 弱点を看破された側は連続追撃結果で「参りました」を選択でき、HP0のSURRENDER KOとして決着する
- 連続追撃でHPを超過したダメージは`OVERKILL`として演出し、ランキングポイントやRATEへは加算しない
- 連続追撃後にHP0なら決着し、それ以外は最大5ラウンドまで継続。同HPなら連続追撃を含む累計画像パワーで判定
- 試合後に両者の本当の弱点・ブラフ・弱点回答・各ラウンドの画像パワー・平均評価を表示

自己紹介、受諾・撤退、デッキ準備、専用チャット、秘密採点、ラウンド進行は参加者だけが読めるRealtime Databaseルームで同期します。チャットは匿名紹介から利用でき、デッキ封印までは相手名・プロフィール画像を表示しません。画像は既存モードと同じCanvas処理で縮小・WebP化し、画像に添付した音声とともにWebRTC DataChannelで対戦相手へ直接転送します。画像・音声本体はFirebaseへ保存せず、途中終了・試合終了・ページ終了時にObject URLを解放します。

### 1on1

- 対戦前に各プレイヤーが画像を5枚選択
- 最大HPは30、最大5ラウンド。サンプル画像を含む場合は1枚につき最大HPが5減り、5枚すべてサンプルならHP5で開始
- 各ラウンドで未使用画像を1枚ずつ秘密裏に選択
- 両者の通信準備完了後に10秒の選択タイマーを開始し、時間切れ時は選択中または未使用の画像を自動ロック
- 両者の選択完了後に画像を交換・公開
- 通常型は気軽に交流できるフリーモードとして、対戦ルーム成立後から試合終了までチャットを常時利用可能
- 相手の画像を1～10点で採点
- 高得点側が勝利し、低得点側は高得点と同じ値のダメージを受ける
- 同点ラウンドは両者ノーダメージ
- 8～9点は `CRITICAL!`、10点は `PERFECT!!`
- 9～10点を受けた画像の持ち主について、事前登録した追撃セリフをラウンド結果へ表示（オンライン1on1では演出のみ）
- HPが0になると敗北
- 5ラウンド終了時は、残りHP、合計獲得点、CRITICAL回数、10点回数の順で判定
- すべて同じ場合は引き分け
- ルーム破棄は勝敗・勝率・連勝数に影響しない

### 2on2

- 待機中の4人が揃うと、ランダムに2人ずつのTEAM A / TEAM Bへ自動分配
- 各プレイヤーが画像を5枚選び、最大5ラウンドすべてで4人全員が1枚ずつ提出
- 各ラウンドは20秒のチーム専用作戦チャット後、10秒の秘密選択を実施
- 選択時間切れ時は、選択中または未使用の画像を自動ロック
- 4枚の公開後、各プレイヤーが相手チームの2枚を別々に採点
- 採点時間は20秒。時間切れ時の未採点画像は5点として自動確定
- 画像得点は相手チーム2人の採点平均、チーム得点はチーム2枚の画像得点平均
- 高得点チームが勝利し、相手の共有HP30へ四捨五入したチーム得点分のダメージ
- 画像得点の平均が8点以上で `CRITICAL!`、相手2人がともに10点なら `PERFECT!!`
- 5ラウンド終了時は、共有HP、累計チーム得点、CRITICAL数、PERFECT数の順で判定
- 作戦中は同じチームの2人だけが読めるチャット、画像公開後は4人共通チャットを使用
- ルーム破棄・切断はノーコンテストとなり、2on2戦績とミッションには影響しない

### 4人バトルロワイヤル

- 待機中の4人が揃うと自動的に個人戦を開始
- 各プレイヤーは `OPENING`・`MIDDLE`・`FINAL` のメイン3枚とリザーブ2枚を登録
- 1試合に1回だけ、そのラウンドのメイン画像を未使用リザーブへ交換可能
- 画像選択は10秒、順位投票は15秒。時間切れ時は選択中の画像・未選択順位を自動ロック
- 画像の持ち主は投票確定まで非表示で、選択・公開・投票中は4人チャットを停止
- 4人生存時は他の3枚を1位3ポイント・2位2ポイント・3位1ポイント、3人生存時は他の2枚を1位2ポイント・2位1ポイントで順位投票
- 毎ラウンド最下位の1人が脱落し、4人→3人→2人→優勝者の順で決着
- 最下位同点の場合は対象者だけがリザーブ画像を提出するサドンデスを実施
- FINALは脱落した2人が優勝画像へ投票。同票なら過去ラウンドの支持率・1位票・ラウンド首位回数・ルーム単位の決定的抽選で判定
- 脱落者はFINAL前まで勝敗に影響しない観客賞を選び、投票確定後の4人共通チャットにも参加
- 結果画面は8秒後に自動進行し、一時切断は30秒間の再接続猶予後に本人だけを失格として対戦を継続
- バトルロワイヤル専用戦績として優勝回数、TOP2回数、対戦数、現在・最高連勝を保存

## オンライン機能

- Firebase Authenticationの匿名アカウント
- トップページに通常型1on1・戦略型1on1・2on2・バトルロワイヤル別の対戦待機中・対戦中人数をリアルタイム表示
- 通常型1on1・戦略型1on1・2on2・バトルロワイヤルを合算するデイリー・ウィークリー・マンスリーの上位50人オンライン総合ランキング（任意参加）
- 4モードの各準備画面とランキング画面に、参加・非参加、Xリンク、コメント受付をまとめた同一の展開式ランキングフォームを表示
- 勝利・バトルロワイヤル1位は3pt、引き分け・バトルロワイヤル2位は1pt、敗北・3位・4位は0ptとして集計
- 日本時間の日付・月曜始まりの週・月を期間キーとして使用し、期間切り替え後も4モード共通の総合RATEを維持
- ランキング参加者同士で、相手1人につき1件・1行80文字以内のコメントを投稿・更新できるプレイヤーコメント
- コメント投稿者と投稿先プレイヤーによる削除、投稿先プレイヤーによるコメント受付ON/OFF
- 毎日0:00（日本時間）に更新される3つのデイリーミッションと、受取式のポイント報酬
- 500PTで買い切りの「トップメッセージ枠」、対戦チャット用の追加リアクション10種、プレイヤー称号18種を購入できるポイントショップ
- トップメッセージ購入者は1行30文字以内のひとことを投稿・編集・非公開にでき、トップページでは新しい5件を8秒ごとに切り替えて表示
- トップメッセージは表示名と投稿時に装備していた称号を表示し、閲覧者はプレイヤー単位で端末内非表示・リセットが可能
- 購入品からリアクションを最大8個、称号を1個装備し、1on1・2on2・バトルロワイヤルのチャットと対戦準備画面へ反映
- ショップ商品は交流・表示専用で、採点・HP・勝敗には影響しない
- 待機中プレイヤーからのランダム1on1マッチング
- 戦略型1on1専用のランダムマッチング、匿名自己紹介、弱点探索チャット、受諾・撤退、デッキ準備、秘密採点、戦績・レート保存
- 通常型1on1のサンプル画像を専用バッジで識別し、実画像枚数・サンプル枚数・開始HPを準備画面と対戦HUDへ表示
- オンライン1on1準備画面で追撃セリフを定型文または1行40文字までの自由記述から登録
- 4人をランダムに2チームへ分ける2on2マッチングと、モード別の2on2戦績・レート
- 4人バトルロワイヤルの自動マッチング、個人脱落、サドンデス、専用戦績
- Firebase Realtime Databaseによるルーム、選択完了、採点、ラウンド進行の同期
- 通常型・戦略型1on1で対戦中の2人だけが読めるチャット
- 2on2のチーム専用作戦チャットと、画像公開後の4人共通チャット
- 勝敗数、引き分け数、現在の連勝、最高連勝の保存
- WebRTC DataChannelによる画像の一時的なP2P転送
- 4モード共通の任意プロフィール画像、対戦中限定表示、他プレイヤー画像の表示／非表示切り替え
- Web Audioによるボタン、決定、選択カウントダウン、クリティカル、パーフェクトの効果音
- ヘッダーの`SE ON / SE OFF`切り替え（設定はブラウザに保存）
- 切断時の待機・対戦予約の自動解放
- 古い対戦提案を次回マッチング開始時に清掃

## 画像とデータの扱い

- 選択画像は1on1では最大1600px、2on2・バトルロワイヤルでは4人転送の通信量を抑えるため最大1280pxに縮小してWebPへ変換します。
- Canvasを経由するため、元画像のEXIFなどの付随情報は引き継ぎません。
- 画像本体と画像IDは、両者がそのラウンドの選択をロックするまで相手へ送りません。
- オンライン画像はWebRTCで対戦相手へ直接送り、Firebase Realtime DatabaseやStorageへ保存しません。戦略型1on1の添付音声も同じDataChannelで直接送り、Firebaseには保存しません。戦略型1on1は通常型1on1とは独立したルームでP2P転送し、2on2・バトルロワイヤルでは各ブラウザが残り3人とP2P接続します。
- プロフィール画像は中央を256px正方形のWebPへ変換して同一ブラウザのIndexedDBだけに保存します。通常型・2on2・バトルロワイヤルは対戦開始後、戦略型は「対戦相手、判明」以降だけ表示し、ランキング・トップページ・最終結果には表示しません。
- 対戦終了、ルーム破棄、ページ終了時に対戦画像と受信したプロフィール画像のURL・Blob参照を解放します。自分のプロフィール画像だけは次回対戦用として端末内に残します。
- Firebaseへ保存するのは匿名UID、表示名、追撃セリフ、トップメッセージ、戦略型1on1の弱点候補・弱点コミット・回答確定後の弱点公開値、モード別戦績、非公開の総合戦績、公開設定時の期間別総合ランキング戦績、ランキングコメント、通常型1on1のサンプル枚数・開始HP、チーム分け、ルーム状態、点数、チャットなどの対戦メタデータです。トップメッセージの公開キーには匿名UIDではなく専用のランダムIDを使用します。戦略型チャットには送信者UID、本文、匿名探索中か対戦中か、ラウンド、送信時刻だけを保存し、プロフィール画像は保存しません。戦略型の待機キューにはUIDと時刻・待機状態だけを置き、弱点番号そのものは両者が回答するまで共有ルームへ保存しません。トップメッセージは1行30文字以内、追撃セリフは1行40文字以内、各チャットとランキングコメントは1行80文字以内に制限し、画面ではHTMLとして実行せず文字列として表示します。
- ミッション進捗、所持ポイント、購入済み商品、装備設定は匿名UIDごとの非公開領域へ保存します。装備中の称号IDだけは、チャット送信時に同じルームの参加者へ共有します。
- トップページの参加状況には人数だけを表示し、閲覧者は人数に含みません。購入者が自分で投稿したトップメッセージだけは表示名・称号とともに公開しますが、匿名UID・ルーム情報・プロフィール画像は表示しません。
- ランキング参加者はプレイヤーネーム・総合RATE・選択期間の総合戦績、合計ポイント、モード別ポイントを公開します。デイリーは毎日0:00、ウィークリーは毎週月曜0:00、マンスリーは毎月1日0:00（日本時間）に新しい期間キーへ切り替わり、データ削除やCloud Functionsは使用しません。本人が個別に許可した場合だけ、Xのユーザー名と自分宛てのコメントも公開します。

ブラウザに表示した画像は、相手によるスクリーンショット、開発者ツール、画面撮影まで技術的に完全防止することはできません。「ゲーム自身が画像ファイルを保存・配布する機能を持たない」という範囲の保護です。

## Firebase構成

このリポジトリはFirebaseプロジェクト `gazostadium` 用に設定済みです。

- Authentication: 匿名ログインを使用
- Realtime Database: 各モードのマッチング、ルーム同期、チャット、戦績に使用
- Storage: 使用しない
- Firebase Hosting: `https://gazostadium.web.app` で本番公開
- クライアント設定: `firebase-config.js`
- セキュリティルールの管理元: `database.rules.json`
- Firebase CLI設定: `.firebaserc` / `firebase.json`（既定プロジェクト `gazostadium`、HostingとRealtime Database Rulesをデプロイ）

Realtime Databaseのルールは、各ユーザーが自分の待機情報・採点・順位投票・モード別戦績・総合戦績・期間別ランキングだけを書けること、各ルームの参加者だけがルーム情報を読めることを制約します。通常型1on1のチャットは対戦ルーム参加者が全対戦フェイズで投稿できます。トップメッセージは商品を所有する本人だけが専用のランダム投稿IDへ書き込めます。戦略型1on1では専用の待機キュー・対戦予約・プロフィール・ルームを分離し、弱点コミット、秘密回答、回答後の弱点公開、連続追撃枚数、降参、画像受信確認、参加者本人によるチャット新規投稿も制限します。総合戦績は4モードの内訳合計、期間別ポイントは勝利数×3＋引き分け数およびモード別ポイント合計と一致する必要があります。ランキングコメントは参加者だけが自分の投稿キーへ書き込め、投稿者本人と投稿先プレイヤーだけが削除できます。Firebaseコンソールでルールを変更する場合は、`database.rules.json`も同じ内容に保ってください。

## Firebaseへデプロイする

Firebase CLIへログイン済みの環境で、次のコマンドを実行するとWeb本体とRealtime Database Rulesをまとめて本番反映できます。

```powershell
npx --yes firebase-tools deploy --only hosting,database --non-interactive
```

本番URL: `https://gazostadium.web.app`

## GitHub Pagesで公開する

1. このフォルダーをGitHubリポジトリへpushします。
2. GitHubのリポジトリで `Settings` → `Pages` を開きます。
3. `Deploy from a branch` を選び、公開ブランチのルートを指定します。
4. 発行されたHTTPS URLへ2つのブラウザ環境からアクセスし、マッチングを確認します。

静的ファイルだけで公開できます。FirebaseのWebクライアント設定は公開される前提の識別情報であり、アクセス制御はRealtime Databaseルールが担当します。

- `strategy.js`: Firebase/WebRTC戦略型1on1の匿名紹介・メイン/リザーブ構築・秘密採点・専用戦績
- `audio.js`: ボタン・クリティカル・パーフェクトの効果音
- `online.js`: Firebase/WebRTC 1on1対戦、4モード共通の総合ランキング、ミッション、ショップ
- `team.js`: Firebase/WebRTC 2on2チーム対戦
- `royale.js`: Firebase/WebRTC 4人バトルロワイヤル
- `firebase-config.js`: Firebase Webクライアント設定
- `database.rules.json`: Realtime Databaseセキュリティルール

## 現時点の制約

- WebRTCは公開STUNサーバーを使用し、TURNサーバーは未設定です。厳しいNATや組織ネットワーク同士ではP2P接続できない場合があります。3接続が必要な2on2・バトルロワイヤルは、1on1より接続失敗の影響を受けやすくなります。
- 匿名アカウント、オンライン戦績、端末保存したプロフィール画像はブラウザのサイトデータに紐づき、サイトデータを消すと新しいユーザー扱いになり、プロフィール画像も再登録が必要です。
- 採点の書き換え先はセキュリティルールで本人の領域に限定していますが、完全な不正対策を行う権威サーバーはありません。
- オンライン総合ランキングはFirebaseだけで動くカジュアル版です。総合RATEと4モードの期間別戦績は各ブラウザから送信されるため、改ざんを完全には防止できません。Cloud Functionsを使用しないため、過去期間の順位確定・書き換え防止・自動削除は行いません。
- サンプル画像のHPハンデもクライアントとRealtime Database Rulesで整合性を確認するカジュアル版です。改造したクライアントがサンプル画像を通常画像として偽装することまでは完全に防止できません。
- ランキングのXリンクは自己申告で、アカウント所有者の本人確認は行いません。Xを公開すると匿名性が下がります。
- ランキングコメントは公開情報です。URLと改行は画面側で拒否し、ルールでも1行80文字以内に制限しますが、自動モデレーションや通報機能はまだありません。
- トップメッセージも公開情報です。URLと改行は画面側で拒否し、ルールでも1行30文字以内に制限します。閲覧者単位の非表示には対応しますが、自動モデレーションや運営向け通報画面はまだありません。
- ミッションとポイントショップもFirebaseだけで動くカジュアル版です。進捗と購入処理はトランザクションで重複を防ぎますが、権威サーバーによる完全な不正対策ではありません。
- 終了済みルームのメタデータは管理用の定期削除をまだ実装していません。画像本体はルームに保存されません。
